# Markdown 任务格式规范

本文件维护项目管理插件当前支持的 Markdown 格式。当前有三类明文格式：

1. 完整任务输入格式：用于快速记录-创建任务、项目进度-批量导入、项目 Markdown 导出。
2. 今日极简完成格式：由今日任务页导出，只用于勾选完成今天已有任务。
3. 全部记录导出格式：由任务总览-项目进度-导出全部记录生成，本质仍是完整任务输入格式，可直接粘贴到快速记录-创建任务中创建或覆盖任务。

## 基本结构

```md
#项目：插件体验示例
- [ ] 开发任务解析器 kind:composite @2026-05-18 09:00-10:30 #parser !high status:doing
  - 解析标题 @09:10-09:40
  - 解析日期 @09:40-10:10
- [ ] 每日回顾输入流程 kind:simple @2026-05-18 20:00-20:20 #review status:todo repeat:daily count:5
- [x] 每周复盘导入流程 kind:simple @2026-05-18 20:30-21:00 #review status:done repeat:weekly count:4 finish:series
```

## 项目分组

| 写法 | 作用 |
| --- | --- |
| `#项目：项目名` | 后续任务归入该项目；项目不存在时会自动创建。 |
| `#项目：` | 后续任务导入为未归属任务。 |
| `#项目：未归属项目` | 等同于未归属任务，主要用于导出文本。 |

项目页批量导入在指定项目内打开时，未显式切换项目的任务会按当前项目处理。

## 创建与覆盖任务

创建或覆盖任务必须提供完整日期和时间段：

```md
- [ ] 任务标题 kind:simple @2026-05-25 09:00-09:30 status:todo
```

规则：

1. `@YYYY-MM-DD HH:mm-HH:mm` 是创建或覆盖任务的必要参数。
2. `repeat:daily` / `repeat:weekly` 仍需要 `count:N` 或 `until:YYYY-MM-DD`。
3. 同项目下同名且同日可匹配的活动任务会被覆盖；否则创建新任务。
4. 时间冲突会自动调整到同日 1 分钟空档。
5. 任务行下方缩进 `> 描述` 可写入任务描述，多行描述会按换行合并。

## 今日极简完成

今日任务页导出的格式只保留项目和标题：

```md
#项目：天力
- [ ] 删除按钮
```

把任务改成已完成后粘回快速记录：

```md
#项目：天力
- [x] 删除按钮
```

规则：

1. 极简 `- [x] 标题` 只会匹配今天已有任务并完成当天发生。
2. 找不到同项目、同标题、今天发生的活动任务时会报错。
3. 极简输入不会创建新任务，也不会覆盖任务字段。
4. 若要新建任务，必须改用完整任务输入格式。

## 参数

| 参数 | 示例 | 作用 |
| --- | --- | --- |
| `kind` / `type` | `kind:simple`、`kind:composite` | 任务类型。`simple` 为普通任务，`composite` 为组合任务。 |
| `@日期 时间` | `@2026-05-18 09:00-10:30` | 任务日期与时间段。创建或覆盖任务时必须填写完整。 |
| `#标签` | `#parser` | 写入任务标签；可写多个。 |
| `!优先级` | `!low`、`!medium`、`!high`、`!urgent` | 写入任务优先级。 |
| `status` | `status:todo`、`status:doing`、`status:blocked`、`status:done` | 写入任务状态。 |
| `repeat` | `repeat:once`、`repeat:daily`、`repeat:weekly`、`repeat:custom` | 重复规则；未写时默认为 `once`。 |
| `count` | `count:5` | 重复次数；用于 `daily` / `weekly`。 |
| `until` | `until:2026-06-30` | 重复结束日期；用于 `daily` / `weekly`。 |
| `dates` | `dates:2026-05-18,2026-05-21` | 自定义发生日期集合；用于 `repeat:custom`。 |
| `done` | `done:2026-05-18,2026-05-21` | 已完成发生日期；全部记录导出会用它保存重复任务进度。 |
| `finish` | `finish:today`、`finish:series` | 仅在完整 `- [x]` 输入时生效；`today` 完成当前日期，`series` 结束重复系列。 |

## 任务描述

任务描述使用缩进引用行：

```md
- [ ] 带描述任务 kind:simple @2026-05-18 09:00-10:30 status:todo
  > 第一行描述
  > 第二行描述
```

导入后描述会保存为两行正文。全部记录导出会用同样格式输出已有任务描述。

## 子任务

组合任务可在任务行下方缩进写轻量子任务：

```md
- [ ] 组合任务 kind:composite @2026-05-18 14:00-15:00 status:todo
  - 子任务一 @14:10-14:30
  - 子任务二 @14:30-14:50
```

只要任务行下存在缩进子项，即使未显式写 `kind:composite`，导入时也会按组合任务处理。

组合任务规则：

1. 组合任务必须有开始时间和结束时间。
2. 轻量子任务可以只写标题；若写了 `@HH:mm-HH:mm`，必须完全落在组合任务时间范围内。
3. 挂入组合任务的真实任务也必须发生在组合任务实例的日期与时间范围内。
4. 表格、看板、甘特图只把组合父任务作为顶层元素展示；真实子任务收纳在父任务摘要内。

## 笔记同步块

全局 Markdown 笔记扫描只读取 `pm:start` 与 `pm:end` 之间的任务块：

```md
<!-- pm:start -->
#项目：插件体验示例
- [ ] 写解析器 kind:simple @2026-05-18 09:00-10:30 #plugin status:todo
<!-- pm:end -->
```

直接粘贴到快速记录或批量导入时，不需要包含这两行注释标记。

## 单次实例编辑

重复任务支持“编辑本次”。它会只覆盖当前日期的标题、描述与时间，不影响未来发生；若要统一改整条重复任务，请改用系列编辑。

## 全部记录导出

“导出全部记录”位于任务总览的“项目进度”标题区，紧邻“新增项目”按钮。

导出内容是明文任务清单，不包含 base64、JSON 数据块或隐藏迁移包：

```md
#项目：插件体验示例
- [ ] 每日回顾输入流程 kind:simple @2026-05-18 20:00-20:20 #review status:todo repeat:daily count:5 done:2026-05-18,2026-05-19
- [x] 每周复盘导入流程 kind:simple @2026-05-18 20:30-21:00 #review status:done repeat:weekly count:4 done:2026-05-18,2026-05-25,2026-06-01,2026-06-08

#项目：未归属项目
- [ ] 临时任务 kind:simple @2026-05-20 09:00-09:30 status:todo
```

规则：

1. 把导出的 Markdown 粘贴到另一台设备的“快速记录 - 创建任务”并提交，即可按普通导入规则创建或覆盖任务。
2. 导入不会替换目标设备全库数据；它只处理文本中的任务行和项目分组。
3. 项目不存在时会根据 `#项目：项目名` 自动创建；`#项目：未归属项目` 会导入为未归属任务。
4. 重复任务的完成进度通过 `done:` 明文日期列表迁移，用户可以在导入前直接编辑。
5. 旧版完整迁移包、`pm-full-transfer` 标记、`pm-json-base64` 数据块和 base64 编码恢复流程已废弃。
6. 今日任务页不提供“导出全部记录”按钮。
