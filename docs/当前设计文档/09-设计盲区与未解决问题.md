# 09. 设计盲区与未解决问题

本轮大改已经解决了“总览不聚焦”和“重复任务缺位”两个核心问题，但仍有一些未完成项。

## 1. 周任务图仍然不是时间网格

当前周任务图已经完成：

1. 日期列重做。
2. 今天高亮。
3. 过去日期灰显。
4. 默认时间智能推算。

但它仍然不是拖拽式时间网格日历。

## 2. 重复任务仍是预生成模式

当前版本采用“创建时生成未来实例”的方案。

优点：

1. 逻辑直观。
2. 今日未完成不会覆盖历史。
3. 系列级操作好理解。

缺点：

1. 长周期重复任务会提前生成较多数据。
2. “修改整个系列从第一天开始”当前没有独立入口。

## 3. 热度图详情仍然偏轻

热度图点击后已经从通知升级为弹窗，但仍然是基础列表，不是完整详情页。

## 4. 项目管理页仍缺少高级交互

当前项目管理页已经增加：

1. 表格视图。
2. 看板视图。
3. 甘特图视图。
4. 思维导图视图。
5. 看板跨列拖拽。
6. 思维导图节点拖拽、缩放和平移。

但还缺少：

1. 筛选。
2. 排序。
3. 看板列内精确排序。
4. 甘特图拖拽排期。
5. 甘特图依赖线可视化编辑。

## 5. 数据迁移未实现

当前版本明确不兼容旧版 JSON 结构，因此：

1. 没有旧数据迁移器。
2. 也没有双格式读写逻辑。

## 6. 全局笔记同步仍是任务块导入

当前全局笔记同步只识别 `<!-- pm:start -->` 与 `<!-- pm:end -->` 包裹的任务块。

尚未实现：

1. 任意 Markdown 任务行自动识别。
2. 笔记与插件任务完整双向镜像。
3. 冲突合并界面。

## 7. 部分配置仍是预留项

配置模型中已经保留 `showCompletedTasks`、`dailyNoteDateFormat`、`weekStartsOn` 和 `heatmapRange`。

当前代码中：

1. 周起始固定按周一计算。
2. 热度图固定展示最近 12 个月。
3. 日记文件名固定使用 `YYYY-MM-DD`。
4. “显示已完成任务”设置会保存，但尚未影响今日页或总览页过滤。
