# 15. 完整迁移包恢复校验

> 本文档保留为阶段性设计记录。涉及 `pm-full-transfer:v1`、完整迁移包恢复与旧恢复约束的描述，均已被更新设计覆盖，请以 `17-明文导出与周任务图简化.md` 为准。

## 1. 最新设计说明

本轮以 `docs/优化建议/bug与优化.md` 为准，修复“导出全部记录”的完整迁移包粘贴到“快速记录 - 创建任务”后报错的问题。

完整迁移包是数据快照恢复协议，不是任务重新创建协议。导入时必须先识别 `pm-full-transfer:v1`，再按快照替换当前项目管理数据。恢复过程只校验快照引用完整性，不重放普通创建、编辑、批量导入时的业务约束。

## 2. 新旧设计差异

旧设计：

1. 迁移包解析后复用任务创建 / 编辑路径中的组合任务边界校验。
2. 如果快照中已有子任务挂在组合任务下，但子任务重复日期超过父组合任务日期范围，恢复会失败。
3. 根目录 `main.js` 与 `project-manager/main.js` 可能不同步，从根目录加载插件时仍可能使用旧导入逻辑。

新设计：

1. 完整迁移包恢复只验证 schema、唯一 ID、项目引用、进度页引用、甘特依赖、思维导图父任务和评语父子关系。
2. 组合任务时间范围、子任务时间范围等创建期约束继续保留在普通任务新建、编辑和普通 Markdown 导入路径中。
3. `npm run build` 同步更新 `project-manager/main.js` 与根目录 `main.js`，避免两个插件入口行为分叉。

## 3. 被删除或废弃的旧逻辑

1. 删除完整迁移包恢复路径中的 `assertCompositeDefinitionValid()` 调用。
2. 删除完整迁移包恢复路径中对子任务与父组合任务日期 / 时间范围的强制校验。
3. 废弃“快照恢复必须满足当前创建期组合任务规则”的旧假设。
4. 废弃根目录构建产物长期停留在旧逻辑的发布方式。

## 4. 新的数据流 / 调用流程

完整导出：

1. `OverviewView` 在项目进度标题区调用 `store.exportAllRecordsAsMarkdown()`。
2. Store 读取当前 `StoreSnapshot`。
3. Store 生成 `FullTransferPackage`，包含配置、项目、进度页、任务、笔记索引和写入历史。
4. Store 将 JSON 编码为 base64，并写入 `pm-full-transfer:v1` Markdown 数据块。

完整导入：

1. `QuickDialogView` 的“创建任务”实时预览调用 `store.previewFormattedTasks()`。
2. Store 优先调用 `parseFullTransferPackage()` 识别迁移包。
3. 识别成功后进入 `previewFullTransferPackage()` 或 `restoreFullTransferPackage()`，不进入普通逐行任务解析。
4. `buildDataStateFromTransferPackage()` 标准化配置、项目、进度页、任务、索引和历史。
5. `assertRestoredSnapshotIntegrity()` 只做快照引用完整性校验。
6. 校验通过后保留目标设备当前 `dataFolder`，替换其余项目管理数据并写入 JSON 文件。

构建发布：

1. `npm run build` 继续以 `src/main.ts` 为唯一源码入口。
2. esbuild 输出 `project-manager/main.js`。
3. 构建结束后同步复制到根目录 `main.js`。

## 5. 关键模块职责

`src/storage/store.ts`

1. 生成、识别、预览和恢复完整迁移包。
2. 区分快照完整性校验与普通任务写入约束。
3. 在普通任务新建、编辑和普通 Markdown 导入时继续执行组合任务边界规则。

`src/views/dialogView.ts`

1. 在快速记录“创建任务”中展示迁移包预览。
2. 提交时复用 `importFormattedTasks()`，由 Store 自动分流迁移包与普通任务文本。

`esbuild.config.mjs`

1. 生成 `project-manager` 发布目录。
2. 保证根目录 `main.js` 与发布目录入口一致。

## 6. 使用方式与开发注意事项

1. 跨设备迁移必须复制整份“项目管理完整迁移包”Markdown。
2. 迁移包导入会替换目标设备当前项目管理数据。
3. 迁移包恢复不能调用普通任务创建 / 编辑约束；否则当前可导出的历史快照可能无法恢复。
4. 新增需要无损迁移的字段时，必须加入 `FullTransferPackage`、标准化逻辑和快照完整性校验。
5. 普通 Markdown 导入仍然必须遵守完整日期时间、组合任务边界和极简完成语义。

## 7. 本次修复的 bug 和优化点

1. 修复完整迁移包被识别后因组合任务边界校验而恢复失败的问题。
2. 修复样例迁移包中重复子任务挂在单次组合任务下时无法导入的问题。
3. 保留普通任务写入路径的组合任务约束，避免放松新任务创建规则。
4. 修复根目录 `main.js` 与发布目录 `main.js` 不同步导致的入口行为不一致问题。
