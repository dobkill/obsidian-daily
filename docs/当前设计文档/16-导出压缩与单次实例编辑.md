# 16. 导出压缩、周任务图与单次实例编辑重构

> 本文档保留为阶段性设计记录。涉及 `pm-full-transfer:v2`、base64 完整迁移包、周任务图“缩略图 / 详细图”双视角的描述，均已被 17 号设计覆盖。当前最新实现以 [17-明文导出与周任务图简化](./17-明文导出与周任务图简化.md) 为准。

本轮直接以 `docs/优化建议/bug与优化.md` 为准，覆盖旧的“历史切片”“局部拉伸时间轴”“pm-full-transfer:v1”设计。

## 1. 目标

本轮要同时解决四类问题：

1. “导出全部记录”体积过大。
2. 空组合任务作为父任务时的可选性问题。
3. 周任务图既不好看，又因为局部拉伸导致时间轴和卡片位置失真。
4. 过去日期无法单独编辑当前发生。

## 2. 最新设计

### 2.1 重复任务模型

当前版本删除历史切片模型，统一改为：

1. 一条重复任务始终只保留一条系列记录。
2. 完成状态记录在 `occurrenceStates`。
3. 单次标题 / 描述 / 时间差异记录在 `occurrenceOverrides`。
4. “编辑整条系列”改系列默认值。
5. “编辑本次”只改当前发生，不影响未来。

这样做的直接收益是：

1. 不再因为每天勾选完成而产生新的历史任务 JSON。
2. 导出时不再出现“一次发生一条任务对象”的膨胀。
3. 周视图、今日页与导出路径都能围绕同一条系列记录工作。

### 2.2 完整迁移包

完整迁移包升级为 `pm-full-transfer:v2`。

核心变化：

1. `occurrenceDates` 不再原样导出，改为连续日期区间数组。
2. 完成记录改为数组元组结构。
3. 单次实例覆盖改为日期 + 差异对象结构。
4. 删除对 `note-task-index.json` 与 `write-history.json` 的导出 / 恢复；它们属于缓存或日志，不是核心业务快照。
5. 恢复时仍保留目标设备当前 `dataFolder`，避免跨设备目录误伤。

### 2.3 周任务图

周任务图拆成两个视角：

1. 缩略图：按天展示卡片列表，优先总览和可读性。
2. 详细图：保留真实时间轴，卡片按真实时间定位，不再做纵向局部拉伸。

这意味着：

1. 缩略图不再强行承担“精确时间轴”职责。
2. 详细图不再为了避免遮挡而破坏时间刻度。

### 2.4 实例编辑

从今日页或周任务图进入实例编辑时：

1. 默认就是“保存本次”。
2. 当前表单只负责编辑本次标题、描述与时间。
3. 若要改整个系列，显式点击“编辑整条系列”再进入系列编辑器。

这样能避免一份表单同时承担“改单次”和“改系列”两套相反语义。

## 3. 新旧设计差异

### 3.1 历史切片

旧设计：

1. 过去日期不可编辑。
2. 每次编辑 / 完成都会把任务切成历史切片和活动切片。

新设计：

1. 过去日期允许单独编辑当前发生。
2. 不再自动切片。
3. 单次差异统一写入 `occurrenceOverrides`。

### 3.2 导出协议

旧设计：

1. `pm-full-transfer:v1`
2. 直接导出完整 `Task` 对象。
3. 同时导出索引与写入历史。

新设计：

1. `pm-full-transfer:v2`
2. 导出压缩后的业务快照。
3. 只保留真正需要恢复的核心数据。

### 3.3 周任务图

旧设计：

1. 单一视图同时承担概览与精确时间轴。
2. 为了防止短任务重叠，对当天做局部纵向拉伸。

新设计：

1. 缩略图负责概览。
2. 详细图负责真实时间定位。
3. 删除局部拉伸逻辑。

## 4. 被删除或废弃的旧逻辑

1. `prepareFutureMutation()` 历史切片写入链路。
2. `sliceTaskToDates()` 任务切片辅助逻辑。
3. “过去日期不可修改 / 删除 / 完成”的核心限制。
4. 周任务图基于局部拉伸的纵向布局策略。
5. `pm-full-transfer:v1` 直接快照导出协议。
6. 完整迁移包中对 `note-task-index.json`、`write-history.json` 的导出恢复。

## 5. 新的数据流 / 调用流程

### 5.1 系列编辑

`TaskModal(系列模式)`  
-> `store.updateTask()`  
-> 重建系列任务  
-> 全量校验组合任务边界与时间冲突  
-> 覆盖原任务  
-> 写文件并刷新视图

### 5.2 单次实例编辑

`TaskModal(实例模式)`  
-> `store.updateTaskOccurrenceDetails()`  
-> 按日期写入 `occurrenceOverrides`  
-> 重新展开实例参与冲突 / 组合校验  
-> 写文件并刷新视图

### 5.3 完整迁移包导出

`OverviewView -> exportAllRecordsAsMarkdown()`  
-> `buildFullTransferPackage()`  
-> `toCompactTask()`  
-> 压缩日期区间 / 完成记录 / 单次覆盖  
-> base64 包装为 `pm-full-transfer:v2`

### 5.4 完整迁移包恢复

`DialogView / BulkImport -> previewFormattedTasks()`  
-> `parseFullTransferPackage()`  
-> `buildDataStateFromTransferPackage()`  
-> 展开日期区间、重建任务  
-> 校验引用完整性  
-> 替换当前项目管理数据

## 6. 关键模块职责

1. `src/storage/store.ts`
   负责系列写入、单次实例覆盖、导出压缩、迁移包恢复和边界校验。
2. `src/components/taskModal.ts`
   明确区分系列编辑与实例编辑模式。
3. `src/views/overviewView.ts`
   负责周任务图缩略 / 详细双视角切换。
4. `src/views/todayView.ts`
   负责从实例上下文进入“编辑本次”。

## 7. 使用方式与开发注意事项

1. 需要改未来规则时，用系列编辑。
2. 需要只改某一天时，用实例编辑。
3. 不要再为“保护过去记录”引入新的切片任务。
4. 新增实例级字段时，优先判断它是否真的属于 occurrence 语义；若属于项目视图或看板语义，应继续保留在系列级。
5. 若继续扩展迁移包，优先扩展压缩结构，不要回退到直接导出完整内部对象。

## 8. 本轮修复与优化点

1. 修复导出包体积过大的根因问题。
2. 修复过去日期无法单独编辑的问题。
3. 修复周任务图时间轴与卡片位置失真的问题。
4. 增加周任务图“缩略图 / 详细图”双视角。
5. 统一实例编辑入口，减少误改整个系列的风险。
