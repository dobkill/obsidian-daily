# 14. 完整迁移与周任务局部拉伸

## 1. 最新设计说明

本轮以 `docs/优化建议/bug与优化.md` 为准，覆盖旧的“全部记录导出只是紧凑台账”设计。

新的“导出全部记录”定位是跨设备完整迁移。导出结果是 Markdown 外壳加 `pm-full-transfer:v1` 数据块，数据块内容为 base64 编码 JSON。另一台设备在“快速记录 - 创建任务”中粘贴整份 Markdown 后，Store 会识别迁移包、校验数据，并替换目标设备当前项目管理数据。

周任务图不再单纯按真实分钟高度摆放卡片。真实时间刻度保持不变，但同一天内如果短任务因为最小卡片高度发生视觉覆盖，只拉伸当天局部 timeline，让相邻卡片有可读间距。

## 2. 新旧设计差异

旧设计：

1. 全部记录导出是人类可读台账。
2. 台账只能表达摘要，无法恢复项目配置、看板顺序、甘特依赖、思维导图坐标、评语、来源、笔记和写入历史。
3. 快速记录只把输入按普通任务文本解析，无法识别全量恢复包。
4. 周任务图按真实时间计算 top，但对卡片使用最小高度，短任务相邻时会互相盖住。

新设计：

1. 全部记录导出是完整迁移包，服务 100% 数据迁移。
2. 快速记录-创建任务优先识别 `pm-full-transfer:v1`，识别成功后进入恢复流程，不走普通任务逐行解析。
3. 项目级批量导入阻断完整迁移包，避免误把全库恢复当成单项目导入。
4. 周任务图保持全局小时高度不变，只在单日列中推开视觉重叠卡片。

## 3. 被删除或废弃的旧逻辑

1. 删除 `exportAllRecordsAsMarkdown()` 中的紧凑台账渲染逻辑。
2. 删除紧凑台账相关的项目分组、组合层级摘要、自然语言状态 / 优先级 / 重复规则输出函数。
3. 废弃“全部记录导出不承担恢复职责”的旧说明。
4. 废弃“真实时间不重叠就一定不会视觉重叠”的周任务图布局假设。

## 4. 新的数据流 / 调用流程

完整导出：

1. `OverviewView` 在项目进度标题区调用 `store.exportAllRecordsAsMarkdown()`。
2. Store 读取 `getSnapshot()`。
3. Store 生成 `FullTransferPackage`：包含 `config`、`projects`、`progressPages`、`tasks`、`noteTaskIndex`、`writeHistory`。
4. Store 将 JSON 编码为 base64，写入 `pm-full-transfer:v1` Markdown 数据块。
5. 页面复制完整 Markdown 到剪贴板。

完整导入：

1. `QuickDialogView` 调用 `store.previewFormattedTasks()` 实时预览。
2. Store 发现 `pm-full-transfer:v1` 标记后，解析 base64 JSON。
3. Store 校验 schema、项目引用、进度页引用、任务 ID、思维导图父任务、甘特依赖、评语父子关系和组合任务边界。
4. 用户提交后，`store.importFormattedTasks()` 调用完整恢复流程。
5. Store 保留目标设备当前 `dataFolder`，替换其余项目管理数据。
6. Store 调用 `flushAll()` 写入全部 JSON 文件，并触发视图刷新。

周任务图：

1. `OverviewView` 先计算周级真实时间范围和固定小时高度。
2. 每天单独构建 display occurrences。
3. 真实时间重叠的任务仍按横向 lane 并排。
4. 同 lane 内如果最小卡片高度导致视觉覆盖，就把后一个卡片向下推开 `WEEK_TIMELINE_CARD_GAP`。
5. 仅该日 timeline 容器高度增加必要空间，整周时间轴高度不被放大。

## 5. 关键模块职责

`src/storage/store.ts`

1. 生成完整迁移包。
2. 识别、预览、校验和恢复完整迁移包。
3. 继续处理普通任务文本导入、今日极简完成和项目级批量导入。
4. 维护数据目录写入、月份任务文件重建和视图刷新。

`src/views/dialogView.ts`

1. 在快速记录-创建任务中展示迁移包预览。
2. 提交迁移包时复用 `importFormattedTasks()`。

`src/components/bulkImportModal.ts`

1. 展示迁移包预览。
2. 在项目级导入中显示阻断问题，避免全库恢复误操作。

`src/views/overviewView.ts`

1. 提供全部记录导出入口。
2. 计算周任务图真实 lane 与单日局部视觉拉伸。

## 6. 使用方式与开发注意事项

1. 跨设备迁移时，复制“导出全部记录”的整份 Markdown，不要只复制可读摘要。
2. 迁移包导入会替换目标设备当前项目管理数据。
3. 迁移包导入保留目标设备当前 `dataFolder`，避免源设备目录名影响目标设备路径。
4. 新增需要无损迁移的字段时，必须纳入 `FullTransferPackage` 和恢复校验。
5. 普通任务导入语法不承担看板、甘特图、思维导图完整恢复职责。
6. 周任务图不要通过增大全局小时高度解决短任务覆盖；只能在单日列局部处理。

## 7. 本次修复的 bug 和优化点

1. 修复“导出全部记录”无法被快速记录-创建任务识别的问题。
2. 修复全部记录导出无法跨设备完整恢复项目、表格、看板、甘特图和思维导图的问题。
3. 修复周任务图中短任务相邻时卡片视觉覆盖的问题。
4. 优化迁移包预览，让用户提交前看到恢复模式、项目数、任务数、历史数和索引数。
5. 删除旧紧凑台账导出逻辑，减少导出职责混乱。
