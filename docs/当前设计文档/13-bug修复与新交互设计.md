# 13. Bug 修复与新交互设计

## 1. 最新设计说明

本轮以 `docs/优化建议/bug与优化.md` 为唯一新需求来源。若旧文档与本文件冲突，以本文件为准。

核心变化：

1. 周任务图同时间段任务按横向 lane 并排展示，不再覆盖。
2. “导出全部记录”只保留在任务总览的“项目进度”标题区，紧邻“新增项目”按钮。
3. 今日任务页删除“导出全部记录”，只保留“导出今日任务”。
4. 全部记录导出改为完整迁移包 Markdown，快速记录-创建任务可直接识别并恢复。
5. 表格、看板、甘特图不再把已挂入组合任务的真实子任务显示为顶层元素。
6. 项目进度页删除“今日自动排程”按钮。
7. 今日任务导出改为极简完成格式；快速记录中极简 `- [x] 标题` 只能完成已有今日任务，不能创建新任务。
8. 任务弹窗新增项目与组合父任务联动；选择组合父任务后自动同步项目与父任务时间范围内的 1 分钟默认时间窗。

## 2. 新旧设计差异

旧设计：

1. 今日导出和项目导出共用完整 checklist 语法。
2. 快速记录会把极简任务行按默认日期补齐并创建新任务。
3. 全部记录导出曾被设计为人类可读台账，无法恢复看板、甘特图、思维导图等完整状态。
4. 项目页三种进度视图直接展示项目下全部任务。
5. 任务弹窗的项目、父组合任务和时间字段互不联动。

新设计：

1. 今日导出只服务“完成已有任务”。
2. 创建或覆盖任务必须提供 `@YYYY-MM-DD HH:mm-HH:mm`。
3. 全部记录导出是跨设备完整迁移协议，承担项目管理数据恢复职责。
4. 项目表格、看板、甘特图展示顶层任务；组合子任务收纳到父任务摘要。
5. 任务弹窗根据项目过滤组合父任务，并自动套用父任务时间窗口。

## 3. 被删除或废弃的旧逻辑

1. 删除今日任务页“导出全部记录”按钮。
2. 删除总览 hero 中的“导出全部记录”按钮。
3. 删除项目进度页“今日自动排程”按钮。
4. 删除紧凑台账导出逻辑，改用 `pm-full-transfer:v1` 迁移包数据块。
5. 删除未使用的今日完整 checklist 导出方法。
6. 废弃“极简任务行可创建任务”的解析语义。

## 4. 新的数据流

今日完成回填：

1. 今日页复制未完成任务为极简 Markdown。
2. 用户把需要完成的行改成 `- [x] 标题`。
3. 快速记录解析项目分组、标题和默认日期。
4. Store 查找同项目、同标题、今天发生的活动任务。
5. 找到则只写完成状态；找不到则抛出阻断错误。

项目三视图展示：

1. 项目页先读取项目下全部任务。
2. 构建 `childrenByParent`，只收纳父任务为组合任务的真实子任务。
3. 表格、看板、甘特图只渲染顶层任务。
4. 父组合任务行 / 卡片 / 甘特左侧信息展示真实子任务摘要和汇总进度。
5. 思维导图继续读取全部任务，保留完整父子结构。

任务弹窗联动：

1. 选择项目后，父组合任务下拉只显示该项目下的组合任务。
2. 选择父组合任务后，任务项目同步为父任务项目。
3. 日期调整为父任务当前可用发生日期。
4. 开始时间取父任务开始时间，结束时间默认为开始后 1 分钟，并保证落在父任务范围内。

完整迁移包：

1. 项目进度标题区点击“导出全部记录”。
2. Store 生成 `pm-full-transfer:v1` Markdown，数据块为 base64 编码 JSON。
3. 用户在另一台设备打开快速记录，切换到“创建任务”，粘贴整份 Markdown。
4. Store 先识别迁移包，再校验 schema、项目引用、任务引用、思维导图父子关系和组合任务边界。
5. 校验通过后替换目标设备当前项目管理数据，写入 `projects.json`、`project-pages.json`、`tasks/*.json`、`note-task-index.json`、`write-history.json`。
6. 导入保留目标设备当前 `dataFolder`，避免跨设备路径名误伤。

周任务图局部拉伸：

1. 全局小时高度保持不变。
2. 每日列先按真实时间分配横向 lane，真实重叠任务仍横向并排。
3. 对同一 lane 内视觉高度会互相压住的相邻任务，只在当天列内向下推开。
4. 仅当天 timeline 容器增加必要高度，左侧整体时间轴不被放大。

## 5. 关键模块职责

`src/storage/store.ts`

1. 判断导入 action：创建、覆盖、完成今日、完成系列。
2. 阻断极简输入创建新任务。
3. 输出今日极简清单和完整迁移包。
4. 识别、预览、校验并恢复 `pm-full-transfer:v1` 完整迁移包。

`src/views/overviewView.ts`

1. 管理全部记录导出入口。
2. 构建项目任务层级视图模型。
3. 周任务图计算真实重叠 lane 与单日局部视觉拉伸。

`src/views/todayView.ts`

1. 今日任务聚焦展示。
2. 导出极简未完成清单。

`src/components/taskModal.ts`

1. 项目与组合父任务联动。
2. 父任务时间窗口自动填充。

## 6. 使用方式与开发注意事项

1. 快速记录中 `- [x] 标题` 是完成命令，不是创建命令。
2. 新建任务必须写完整日期和时间段。
3. 项目页三视图新增数据展示时应使用顶层任务集合，不要直接遍历项目全部任务。
4. 思维导图是完整层级视图，不应套用顶层任务过滤。
5. 全部记录导出是当前唯一的无损迁移格式；导入会替换目标设备当前项目管理数据。
6. 不要再恢复紧凑台账导出；它无法表达看板、甘特图和思维导图状态。

7. 删除项目页今日自动排程按钮。
