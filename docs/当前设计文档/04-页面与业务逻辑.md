# 04. 页面与业务逻辑

## 任务总览

任务总览页由 `OverviewView.render()` 渲染，顶部标题固定为“任务总览”，切换按钮默认显示“任务总览 / 项目进度”。任务总览 Tab 读取 `ProjectManagementStore.getSnapshot()` 中的 `occurrences` 和 `tasks`，不写入业务数据。

页面从上到下展示：

1. 今日统计卡片：今日任务、今日逾期、当前进行中、阻塞任务、本周完成。
2. 近 30 天趋势：按日期统计每日任务实例总数和每日完成实例数。
3. 今日时间线：只展示今天 `doing` 且未完成的普通任务和子任务；组合容器只作为有子任务时的分组展示。
4. 周任务图：按当前周七天展示任务实例，支持新增任务、勾选完成、编辑单次实例。
5. 项目热力趋势：展示近 1 年完成任务实例热度。

统计口径：

1. 今日任务：今天发生且状态为 `doing` 的普通任务和子任务数量。
2. 今日逾期：状态为 `doing`、尚未完成，且日期早于今天或今天已超过结束时间的普通任务和子任务数量。
3. 当前进行中：今天状态为 `doing`、尚未完成，且当前时间位于开始时间和结束时间之间的普通任务和子任务数量。
4. 阻塞任务：状态为 `blocked` 且系列尚未全部完成的普通任务和子任务数量。
5. 本周完成：完成时间位于本周一至本周日之间的任务实例数量。

`todo` 和 `blocked` 的未完成实例不计入今日任务、今日逾期和当前进行中。

## 今日任务页

今日任务页读取 `getTasksForDate(today)`，并通过 `buildCompositeDisplayOccurrences()` 将组合任务和挂入组合任务的真实子任务汇总展示。

页面分为三栏：

1. 未完成：只显示状态为 `doing` 且未完成的普通任务和子任务；空容器不显示在这里。
2. 留意：显示空容器，以及状态为 `todo` 或 `blocked` 且未完成的普通任务和子任务。
3. 已完成：显示今天已完成的任务实例。

操作：

1. 普通任务勾选调用 `updateTaskOccurrenceCompletion()`。
2. 组合任务的子任务勾选调用子任务自己的 `updateTaskOccurrenceCompletion()`。
3. 编辑任务打开 `TaskModal`。
4. 导出今日任务调用 `exportTasksAsMinimalCompletionText()`，只复制今日 `doing` 且未完成的可执行任务。

## 任务弹窗

`TaskModal` 负责创建和编辑任务。

普通任务和子任务支持：

1. 状态：`todo`、`doing`、`blocked`、`done`。
2. 重复类型：每日此时、每周此时、每月此时。
3. 快捷选择：单次任务、永久重复。
4. 执行足额次数开关。

组合任务只作为容器：

1. 不显示状态、优先级、标签、重复规则和执行足额次数。
2. “子任务”区域展示添加子任务按钮和已挂入的子任务列表。
3. 保存后的组合任务可直接创建挂入该组合任务的普通任务。

选择“挂入组合任务”时，任务会作为该组合任务的子任务，并保留自己的状态、重复规则、完成记录和导图评语能力。

## 项目进度页

项目进度页按 `ProgressPage` 顺序展示项目。每个项目有四个视图：

1. 表格：展示任务名称、状态、优先级、重复、计划、完成、描述和操作；不显示标签列。不同状态使用不同颜色。组合任务显示为容器。
2. 看板：只按普通任务和子任务的状态分列展示；组合容器不作为看板卡片参与统计。
3. 甘特图：只显示组合容器；容器时间范围由挂入子任务的日期范围决定，空容器范围为 0。
4. 思维导图：按 `viewState.mindmap` 和 `mindmapComments` 展示项目、任务、子任务和评语节点。

项目“总任务”统计只计算普通任务和子任务，与组合容器数量无关。

## 快速记录页

快速记录页有四个目标：

1. 写日记：写入每日文件或汇总文件。
2. 追加笔记：追加到任意 Markdown 文件。
3. 创建任务：解析数据迁移 JSON、今日完成极简 Markdown、新任务计划复杂 Markdown。
4. 补充导图：普通任务、子任务和评语节点都可以被选择为补充目标。

## 设置页

设置页顶部展示三种创建格式。每种格式都有说明、可编辑示例文本框和复制按钮。文本框内容不持久化，重新打开设置页恢复默认示例。
