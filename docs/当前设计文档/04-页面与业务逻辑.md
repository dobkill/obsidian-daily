# 04. 页面与业务逻辑

## 任务总览

任务总览页由 `OverviewView.render()` 渲染，顶部标题固定为“任务总览”，切换按钮默认显示“任务总览 / 项目进度”。任务总览 Tab 读取 `ProjectManagementStore.getSnapshot()` 中的 `occurrences` 和 `tasks`，不写入业务数据。

页面从上到下展示：

1. 今日统计卡片：今日任务、今日逾期、当前进行中、阻塞任务、本周完成。
2. 近 30 天趋势：按日期统计每日任务实例总数和每日完成实例数。
3. 今日时间线：读取今天的任务实例，组合任务会合并显示父任务与挂入的真实子任务，按开始时间排序，并标记当前时段和已超时任务。时间线面板高度固定为与左侧趋势图一致，任务行在面板内纵向滚动。用户滚动后 5 秒没有继续操作时，自动定位到当前进行中的任务；没有当前任务时定位到今日逾期任务。
4. 周任务图：按当前周七天展示任务实例，支持新增任务、勾选完成、编辑单次实例；组合任务只显示挂入组合任务的子任务名，降低卡片高度和信息密度。
5. 项目热力趋势：展示近 1 年（365 天）完成任务实例热度。区域使用 GitHub contribution graph 风格：顶部按月份显示月份标签，左侧为 7 行星期标签（周一至周日），主体为 7 行 × 53-54 列的格子矩阵，每列代表一周，每个格子代表一天；列宽会均匀分配并铺满底部区域。格子颜色深浅表示完成数量（0-4 级紫色热度），右下角显示"少/多"紫色色阶图例；今天所在日期格显示额外描边。点击任一日期格打开当天任务明细。

统计口径：

1. 今日任务：今天发生且尚未完成的展示任务数量。
2. 今日逾期：未完成且日期早于今天，或今天已超过结束时间的任务实例数量。
3. 当前进行中：今天尚未完成且当前时间位于开始时间和结束时间之间的展示任务数量。
4. 阻塞任务：状态为 `blocked` 且系列尚未全部完成的任务数量。
5. 本周完成：完成时间位于本周一至本周日之间的任务实例数量。

## 今日任务页

今日任务页读取 `getTasksForDate(today)`，并通过 `buildCompositeDisplayOccurrences()` 将组合任务和挂入组合任务的真实子任务汇总展示。

操作：

1. 普通任务勾选调用 `updateTaskOccurrenceCompletion()`。
2. 组合任务的子任务勾选调用子任务自己的 `updateTaskOccurrenceCompletion()`。
3. 编辑任务打开 `TaskModal`。
4. 导出今日任务调用 `exportTasksAsMinimalCompletionText()`，只复制极简今日完成格式；组合任务下的未完成项按真实子任务导出，不导出父容器占位行。

组合任务在今日任务页、周任务图和任务弹窗内使用 `compositeTaskCards` 展示真实子任务。今日任务页保留子任务重复规则、时间、编辑和删除操作；周任务图使用紧凑模式，只显示子任务名。子任务进度汇总到组合父任务展示中。

## 任务弹窗

`TaskModal` 负责创建和编辑任务。选择“挂入组合任务”时，任务项目跟随组合父任务，日期落到父任务发生日期内，开始时间和结束时间直接采用父组合任务窗口；保存时由 `ProjectManagementStore` 统一校验时间格式、父子日期关系和父子时间范围。

编辑组合任务时，“子任务”区域展示：

1. “添加子任务”按钮，保存后的组合任务可直接创建挂入该组合任务的普通任务。
2. 已挂入该组合任务的子任务列表。点击子任务的编辑按钮会打开该任务自己的编辑弹窗。

## 项目进度页

项目进度页按 `ProgressPage` 顺序展示项目。每个项目有四个视图：

1. 表格：展示任务主字段，支持新增、编辑、删除、批量导入和项目 Markdown 导出。批量导入仅支持"新任务计划复杂 Markdown"格式，导入时 `#项目：` 必须匹配当前项目；同项目内同名任务覆盖计划字段并保留既有完成记录。导出 Markdown 输出当前项目的"新任务计划复杂 Markdown"格式文本，包含表格、看板、甘特图和思维导图所需状态，但不包含完成记录。
2. 看板：按 `status` 和 `viewState.board` 展示任务。
3. 甘特图：按日期、时间、`viewState.gantt` 展示任务。
4. 思维导图：按 `viewState.mindmap` 和 `mindmapComments` 展示任务节点与评语节点。

“导出全部记录”位于项目进度页标题区，复制数据迁移 JSON。

## 快速记录页

快速记录页有四个目标：

1. 写日记：写入每日文件或汇总文件。
2. 追加笔记：追加到任意 Markdown 文件。
3. 创建任务：解析数据迁移 JSON、今日完成极简 Markdown、新任务计划复杂 Markdown。
4. 补充导图：把内容追加到任务或评语正文，或逐行创建评语子节点。

## 设置页

设置页顶部展示三种创建格式。每种格式都有说明、可编辑示例文本框和复制按钮。文本框内容不持久化，重新打开设置页恢复默认示例。
