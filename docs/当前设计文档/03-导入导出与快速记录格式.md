# 03. 导入导出与快速记录格式

快速记录-创建任务支持三类格式。三类格式职责独立，解析时按数据迁移 JSON 优先，其次解析 Markdown。

## 数据迁移 JSON

“导出全部记录”调用 `ProjectManagementStore.exportAllRecordsAsDataMigrationJson()`，再由 `buildDataMigrationJson()` 生成 JSON。

```json
{
  "schema": "obsidian-project-management/data-migration",
  "version": 2,
  "exportedAt": "2026-05-26T12:00:00+08:00",
  "projects": [],
  "progressPages": [],
  "tasks": []
}
```

`tasks` 中每条任务使用紧凑记录：

1. `daily/weekly/monthly` 可由 `date + recurrence + recurrenceCount/recurrenceUntil` 推导时，不导出完整日期数组。
2. 删除或补充过的实例写入 `occurrencePlan.exclude` 和 `occurrencePlan.include`。
3. 空数组、空描述、默认状态和默认视图状态不导出。
4. 完成状态、单次实例覆盖、执行足额次数、看板、甘特图、思维导图状态、评语、任务笔记和来源链接保留在 JSON 内。
5. 组合任务的子任务通过独立任务的 `viewState.mindmap.parentTaskId` 表达，不使用轻量 `subtasks`。

导入流程：

`QuickDialogView / BulkImportModal`
-> `previewFormattedTasks()`
-> `parseDataMigrationText()`
-> `buildDataMigrationImportPreview()`
-> `importDataMigrationPackage()`
-> 项目与任务 ID 映射
-> 引用重写
-> 业务校验
-> 写回项目、项目页和任务文件。

## 今日完成极简 Markdown

今日任务页导出的文本只用于完成今天已有任务：

```md
#项目：英语四级冲刺
- [x] 每日背词
- [ ] 听力精听
```

规则：

1. `- [x] 标题` 只匹配同项目、同标题、今天发生的任务并完成当天实例。
2. 找不到今日已有任务时阻止导入。
3. 极简格式不会创建新任务，也不会覆盖任务字段。

## 新任务计划复杂 Markdown

新计划使用 `+ 任务：`、`+ 组合：` 或缩进的 `+ 子任务：`：

```md
#项目：英语四级冲刺
+ 任务：搭建复习看板 @2026-05-27 09:00-10:30 !high status:doing
+ 任务：整理错题索引 @2026-05-28 status:todo
+ 组合：拆解每日背词 @2026-05-27 12:00-12:40 board:todo:0 gantt:order=20 mindmap:order=20,expanded
  + 子任务：复习昨天错词 @2026-05-27 12:00-12:10 status:todo repeat:daily count:5 board:todo:0 gantt:order=21 mindmap:order=10,expanded
  + 子任务：新增 30 个高频词 @2026-05-27 12:10-12:30 status:doing repeat:monthly count:5 board:doing:0 gantt:order=22 mindmap:order=20,expanded
  > 每天完成后可在今日任务页勾选。
```

规则：

1. `任务：`、`组合：`、`子任务：` 冒号后可以直接写标题，也可以先写一个或多个空格。
2. `+ 任务：` 表示普通计划任务，必须提供显式 `@YYYY-MM-DD`；可以只写日期表示未排期，也可以写完整 `@YYYY-MM-DD HH:mm-HH:mm`。
3. `+ 组合：` 表示容器，容器不支持状态和重复规则。
4. 缩进的 `+ 子任务：` 会创建独立普通任务并挂入上方组合任务。
5. 快速记录中 `#项目：项目名` 会自动匹配或新建项目；项目进度页批量导入只接受当前项目名，不匹配会阻止导入。
6. 支持 `!优先级`、`status`、`repeat`、`count`、`until`、`board`、`gantt`、`deps`、`parent`、`mindmap`。
7. `repeat` 只支持 `daily`、`weekly`、`monthly`；不写 `repeat` 时按单次任务处理，即 `daily + count:1`。
8. 缩进 `>` 写任务描述；写在子任务行下方时归入该子任务。
9. 新任务计划复杂 Markdown 不保存完成记录。同项目同名任务会覆盖计划字段，同时保留既有完成状态记录；没有同名任务则创建。

视图状态参数：

| 参数 | 示例 | 作用 |
| --- | --- | --- |
| `board` | `board:doing:10` | 看板列与排序。 |
| `gantt` | `gantt:order=20,locked,milestone` | 甘特图排序、锁定和里程碑。 |
| `deps` | `deps:%E4%BB%BB%E5%8A%A1A|%E4%BB%BB%E5%8A%A1B` | 甘特依赖，任务标题使用 URL 编码。 |
| `parent` | `parent:%E7%BB%84%E5%90%88%E4%BB%BB%E5%8A%A1A` | 挂入同项目下指定组合任务。 |
| `mindmap` | `mindmap:order=20,expanded,x=280,y=120` | 思维导图排序、展开状态和坐标。 |
