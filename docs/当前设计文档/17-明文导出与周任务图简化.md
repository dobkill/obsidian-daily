# 17. 明文导出与周任务图简化

本轮以 `docs/优化建议/bug与优化.md` 为最高优先级，覆盖旧的 base64 完整迁移包和周任务图详细图设计。

## 1. 问题分析

### 1.1 导出全部记录无法满足新需求

旧实现中，任务总览页的“导出全部记录”调用 `ProjectManagementStore.exportAllRecordsAsMarkdown()`，生成 `pm-full-transfer:v2` Markdown 外壳，并把压缩 JSON 通过 `pm-json-base64` 数据块保存。快速记录-创建任务入口会先调用 `parseFullTransferPackage()`，识别后走 `restoreFullTransferPackage()` 替换当前全库数据。

这个链路的问题是：

1. 导出内容不是明文任务清单，用户无法方便手动修改标题、日期、时间、标签或重复规则。
2. 导入语义是“替换全部快照”，不是“通过创建任务入口创建任务”。
3. base64 数据块与新需求“不能使用 base64 编码，必须明文”直接冲突。
4. 为兼容旧迁移包保留解析 / 恢复分支会继续暴露旧设计，因此需要删除而不是隐藏。

涉及模块：

1. `src/views/overviewView.ts`：导出按钮入口。
2. `src/storage/store.ts`：导出、预览、导入、旧迁移包解析和恢复逻辑。
3. `src/views/dialogView.ts`、`src/components/bulkImportModal.ts`：导入预览 UI。
4. `src/utils/markdownGuide.ts` 与 `docs/markdown——type.md`：语法说明。

### 1.2 周任务图详细图需要删除

旧实现中，`OverviewView` 保存 `weekViewMode: "compact" | "detail"`，页面渲染“缩略图 / 详细图”切换。详细图通过 `renderWeekDetailBoard()` 构建时间轴、小时刻度、绝对定位卡片和 lane 计算。

这个链路的问题是：

1. 新需求明确要求删除详细图视图。
2. 继续保留状态、CSS 或布局函数会让产品形态和维护入口仍然存在旧设计。
3. 周任务图当前核心价值是按天概览、快速新增、实例完成与实例编辑，不再承担精确时间轴职责。

涉及模块：

1. `src/views/overviewView.ts`：周任务图状态、切换 UI、详细图渲染和布局算法。
2. `styles.css`：详细图时间轴、定位卡片和切换控件样式。

## 2. 最新设计

### 2.1 全部记录导出

“导出全部记录”现在导出纯 Markdown 任务清单，格式与快速记录-创建任务、项目批量导入、项目 Markdown 导出完全一致。

导出示例：

```md
#项目：插件体验示例
- [ ] 每日回顾 kind:simple @2026-05-18 20:00-20:20 #review status:todo repeat:daily count:5 done:2026-05-18,2026-05-19
  > 复盘当天输入流程和阻塞点
- [x] 周复盘 kind:simple @2026-05-18 20:30-21:00 #review status:done repeat:weekly count:4 done:2026-05-18,2026-05-25,2026-06-01,2026-06-08

#项目：未归属项目
- [ ] 临时任务 kind:simple @2026-05-20 09:00-09:30 status:todo
```

核心规则：

1. 不再生成 `pm-full-transfer` 标记。
2. 不再生成 `pm-json-base64` 数据块。
3. 不再执行“替换全部”恢复。
4. 导入时始终走普通任务解析、预览和创建 / 覆盖流程。
5. 重复任务完成进度使用 `done:YYYY-MM-DD,YYYY-MM-DD` 明文参数表达。
6. 任务描述使用任务行下缩进 `> 描述` 明文表达。

### 2.2 周任务图

周任务图只保留单一紧凑视图：

1. 按周展示 7 个日期列。
2. 每列按任务开始时间排序展示任务卡片。
3. 支持在某日新增任务。
4. 支持勾选实例完成状态。
5. 支持从任务卡进入实例编辑。
6. 支持展示组合任务轻量子项和挂入组合任务的真实子任务摘要。

详细时间轴视图、小时刻度、绝对定位卡片、lane 计算和对应 CSS 均已删除。

## 3. 数据流 / 调用流程

### 3.1 全部记录导出

`OverviewView 导出全部记录`
-> `ProjectManagementStore.exportAllRecordsAsMarkdown()`
-> `renderTaskListForImport()`
-> 按项目分组输出 `#项目：...`
-> `renderTaskSeriesForExport()`
-> 输出 checklist 任务行、描述行、组合任务子项与 `done:` 完成日期
-> 复制到剪贴板

### 3.2 明文任务导入

`QuickDialogView / BulkImportModal`
-> `previewFormattedTasks()`
-> `parseFormattedTaskText()`
-> `buildTaskImportPreview()`
-> 用户提交
-> `importFormattedTasks()`
-> `applyImportTask()`
-> `createTask()` 或 `updateTask()`
-> 写入 JSON 并刷新视图

### 3.3 周任务图

`OverviewView.renderActivityTab()`
-> `renderWeekCompactBoard()`
-> `buildCompositeDisplayOccurrences()`
-> `renderWeekTaskCard()`
-> 勾选 / 编辑 / 新增回到 Store 的实例级方法

## 4. 新旧设计差异

旧设计：

1. `导出全部记录` 是 base64 完整迁移包。
2. 快速记录导入迁移包后替换目标设备当前项目管理数据。
3. 导入预览展示“恢复模式 / 替换全部 / 项目数 / 任务数”。
4. 周任务图存在“缩略图 / 详细图”双视图。
5. 详细图维护独立时间轴、小时刻度和 lane 布局。

新设计：

1. `导出全部记录` 是可读、可编辑、可直接导入的 Markdown 任务清单。
2. 快速记录始终按创建任务语法导入，不再执行快照恢复。
3. 导入预览只展示任务解析、创建、覆盖、完成等普通导入结果。
4. 周任务图只保留单一紧凑视图。
5. 时间轴详细图和相关布局算法彻底删除。

## 5. 被删除或废弃的旧逻辑

1. `FULL_TRANSFER_SCHEMA`、`FULL_TRANSFER_START`、`FULL_TRANSFER_END`、`FULL_TRANSFER_CODE_BLOCK`。
2. `parseFullTransferPackage()`、`buildFullTransferPackage()`、`restoreFullTransferPackage()`。
3. `CompactTask` / `FullTransferPackage` 等旧迁移包类型。
4. base64 编码、解码和换行包装函数。
5. 导入预览中的完整迁移包摘要 UI。
6. `weekViewMode`、周任务图视角切换控件。
7. `renderWeekDetailBoard()`、周任务图时间轴 range / position / lane 计算函数。
8. `.pm-week-timeline-*`、`.pm-week-time-axis`、`.pm-week-task.is-positioned` 等详细图样式。

## 6. 关键模块职责

1. `ProjectManagementStore`
   负责明文任务导出、任务文本解析、导入预览、创建 / 覆盖任务、重复任务完成日期写入。
2. `OverviewView`
   负责导出入口、单一周任务图渲染、实例级周任务交互。
3. `QuickDialogView`
   负责快速记录-创建任务输入、实时预览和提交。
4. `BulkImportModal`
   负责项目页批量导入，复用同一套明文任务语法。
5. `markdownGuide` 与 `docs/markdown——type.md`
   负责说明当前可导入语法，不再描述旧迁移包。

## 7. 使用方式与开发注意事项

1. 跨设备迁移任务时，复制“导出全部记录”的明文内容，到目标设备快速记录-创建任务中提交。
2. 导入前可以直接编辑任务行；不要再手动构造 `pm-full-transfer` 或 base64 数据块。
3. 任务描述可通过缩进 `>` 多行表达；导入后会按换行合并。
4. `done:` 只表示完整完成的发生日期；组合任务某一天只完成部分轻量子项时，当前明文导入语法不会逐项表达。
5. 新增导出字段时，优先扩展可读任务语法，避免重新引入隐藏 JSON、base64 或替换全库恢复流程。
6. 周任务图后续优化应围绕按天概览、卡片密度、筛选和实例操作，不应恢复详细时间轴视图。

## 8. 本次修复与优化点

1. 修复“导出全部记录”不能作为明文任务清单直接导入创建任务的问题。
2. 删除 base64 完整迁移包和替换全部恢复链路。
3. 新增 `done:` 明文完成日期参数，保留重复任务迁移时的完成进度。
4. 新增缩进 `>` 描述行，保留任务描述正文。
5. 删除周任务图详细图视图、入口、布局函数和样式。
6. 更新导入预览、快速记录提示、内置 Markdown 语法说明和设计文档。
