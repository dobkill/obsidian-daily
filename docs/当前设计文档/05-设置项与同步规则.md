# 05. 设置项与同步规则

## 设置项

设置页写入 `PluginConfig`，变更后通过 `ProjectManagementStore.setConfig()` 落盘。

主要设置：

1. 数据目录路径。
2. 任务总览、项目进度、快速记录页名称。
3. 日记文件夹、日期格式、日记保存方式和汇总文件路径。
4. 笔记追加头和日记插入头。
5. 最近笔记数量。
6. 快速记录默认目标。
7. 时间粒度、默认任务时长、默认开始时间。
8. 是否显示已完成任务。

## 数据目录切换

数据目录必须是 Vault 内相对路径。目标目录已有有效插件数据时直接加载；目标目录不存在或为空时创建新数据文件；目标目录结构异常时阻止读取并提示。

允许的目录内容：

1. `config.json`
2. `projects.json`
3. `project-pages.json`
4. `note-task-index.json`
5. `write-history.json`
6. `markdown-format-guide.md`
7. `tasks/`

## 笔记同步

全库笔记同步只读取 Markdown 文件中的任务块：

```md
<!-- pm:start -->
#项目：英语四级冲刺
+ 任务：听力精听 @2026-05-27 20:00-20:30 #listen status:todo
<!-- pm:end -->
```

同步流程：

1. 扫描 `pm:start` / `pm:end` 之间的文本。
2. 用复杂 Markdown 解析任务计划。
3. 通过来源文件路径和内容 hash 匹配已有任务。
4. 新内容创建或更新任务。
5. 已消失的来源标记为 `missing`，不直接删除任务。

## 写入历史

批量导入、自动排程、笔记同步和快速记录追加文件会写入 `write-history.json`。历史记录用于最近文件候选和操作回溯。
