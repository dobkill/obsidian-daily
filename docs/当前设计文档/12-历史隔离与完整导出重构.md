# 12. 历史隔离与完整导出重构

## 1. 最新设计说明

本次重构以后，新需求是唯一标准：过去日期的任务记录不可变，今天和未来的任务计划可继续编辑。

核心变化：

1. 编辑、删除、提前结束重复任务时，只处理今天和未来的发生日期。
2. 过去发生日期保留为历史切片，旧标题、旧时间、旧状态、旧说明、旧视图状态和旧评语节点不被覆盖。
3. 全部记录导出改为完整迁移包 Markdown，由快速记录-创建任务识别并恢复全量项目管理数据。
4. 组合任务拥有明确时间范围，轻量子任务和挂入的真实子任务必须落在父组合任务范围内。
5. 快速记录目标切换改为 tab，笔记追加和日记写入支持可配置插入头。
6. 周任务图增加左侧时间轴和背景时间线。

## 2. 新旧设计差异

旧设计：

1. 一条重复任务直接保存全部过去和未来发生日期。
2. 修改系列会重算整条 `occurrenceDates`，过去发生记录可能被新配置覆盖。
3. 删除系列会删除整条任务。
4. 到本次为止结束重复会把过去发生补标为完成。
5. 逾期单次任务会在加载后自动顺延到今天。
6. Markdown 导出没有区分“创建任务”和“完成已有任务”，极简 `- [x]` 可能误创建新任务。
7. 快速记录顶部是四个大目标卡片。

新设计：

1. Store 写入前按今天切分历史切片和活动切片。
2. 过去发生不可编辑、不可删除、不可改时间、不可重新标记完成。
3. 删除系列只删除今天和未来；如果存在过去发生，历史切片保留。
4. 提前结束只标记今天到本次为止的活动发生，过去状态保持原样。
5. 不再自动顺延逾期单次任务。
6. 全部记录导出是跨设备无损迁移协议；创建任务和今日完成继续使用不同输入语义。
7. 快速记录目标切换使用 tab。

## 3. 被删除或废弃的旧逻辑

1. 删除 `normalizeDeferredSingleTasks` 入口和逾期单次任务自动顺延逻辑。
2. 删除旧 `completedOccurrences` 兼容读取逻辑。
3. 废弃“整条重复任务无条件重写”的编辑语义。
4. 废弃“删除任务等于删除全部历史记录”的语义。
5. 废弃把过去发生补标完成的提前结束语义。

## 4. 新的数据流

编辑任务：

1. 页面提交 `TaskInput`。
2. Store 标准化输入并生成候选任务。
3. Store 按今天拆分原任务过去日期和候选任务今天/未来日期。
4. 过去日期写入历史切片，今天/未来写入活动切片。
5. 校验时间冲突、组合任务边界和思维导图父子关系。
6. 写入对应月份 JSON 并触发视图刷新。

删除任务：

1. Store 检查任务是否存在今天或未来发生。
2. 若不存在，拒绝删除。
3. 若存在过去日期，保留历史切片。
4. 删除今天和未来发生。

全部记录导出：

1. Store 读取当前快照。
2. 生成 `pm-full-transfer:v1` Markdown 外壳。
3. 将 `config`、`projects`、`progressPages`、`tasks`、`noteTaskIndex`、`writeHistory` 编码为 base64 JSON 数据块。
4. 页面复制完整迁移包到剪贴板。
5. 另一台设备通过快速记录-创建任务导入时，Store 校验并替换目标设备当前项目管理数据。

快速记录写入 Markdown：

1. 根据目标选择 `noteAppendHeader` 或 `dailyNoteHeader`。
2. 按配置生成标题行。
3. 将标题行和正文追加到目标 Markdown 文件。
4. 写入记录进入 `write-history.json`，用于最近 10 个操作文件。

## 5. 关键模块职责

`src/storage/store.ts`

1. 任务创建、编辑、删除、完成、排程的唯一业务规则入口。
2. 历史切片、组合任务时间边界、时间冲突、完整迁移包导出 / 导入、语法说明导出。

`src/components/taskModal.ts`

1. 任务创建和编辑 UI。
2. 分区展示基础信息、时间安排、归属状态、重复规则和组合轻量项。
3. 轻量项支持标题和可选时间。

`src/views/overviewView.ts`

1. 总览主入口。
2. 在项目进度标题区提供全部记录导出按钮。
3. 周任务图渲染时间轴。
4. 项目页维持表格、看板、甘特图、思维导图四视图。

`src/views/todayView.ts`

1. 今日任务聚焦操作。
2. 提供今日未完成极简导出；不再提供全部记录导出。

`src/views/dialogView.ts`

1. 快速记录 tab。
2. 日记和笔记追加头格式化。
3. 最近 10 个操作文件稳定展示。

`src/settings.ts`

1. 插件配置入口。
2. 插入头格式配置。
3. Markdown 语法说明文件导出。

## 6. 使用方式与开发注意事项

1. 需要修改过去记录时，不应通过编辑旧任务实现；过去记录是审计数据。
2. 组合任务必须填写开始和结束时间。
3. 挂入组合任务的真实子任务必须有时间，并且必须落在父组合任务范围内。
4. 全部记录导出是完整迁移包，不再使用旧的人类可读台账。
5. 迁移包导入会替换目标设备当前项目管理数据，并保留目标设备当前 `dataFolder`。
6. 新增写入逻辑必须通过 Store，不要在视图层直接改任务数组。

## 7. 本次修复与优化点

1. 修复编辑重复任务污染过去记录的问题。
2. 修复删除任务会删除过去记录的问题。
3. 修复提前结束重复任务会改写过去完成状态的问题。
4. 删除逾期单次任务自动顺延导致历史记录变化的问题。
5. 重构全部记录导出，删除紧凑台账逻辑，改为可被快速记录导入的完整迁移包。
6. 新增组合任务时间范围和子任务范围校验。
7. 优化任务弹窗布局。
8. 优化快速记录目标切换和最近文件展示。
9. 新增笔记追加头和日记插入头设置。
10. 新增设置页 Markdown 语法说明导出。
11. 优化周任务图时间轴展示。
