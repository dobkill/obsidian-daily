# 01. 项目概述

本插件是 Obsidian 内的项目与任务管理工具。当前版本以一份任务数据源支撑任务总览、今日任务、项目进度和快速记录四类视图。

## 当前能力

1. 任务总览展示今日状态统计、近 30 天趋势、可滚动今日时间线、周任务图和 GitHub 风格近 1 年（365 天）项目热力图。
2. 今日任务页聚合当天任务实例，支持完成普通任务、完成组合任务项、编辑任务和导出今日极简完成清单。
3. 项目进度页按项目展示表格、看板、甘特图和思维导图；这些视图都读写同一条 `Task` 记录。
4. 快速记录页支持写日记、追加到任意 Markdown 文件、创建任务和补充思维导图评语。
5. “导出全部记录”输出数据迁移 JSON，可通过快速记录-创建任务恢复项目、项目页和完整任务记录。

## 当前架构

1. `src/main.ts` 负责插件生命周期、视图注册、命令注册、设置同步和后台笔记扫描。
2. `src/storage/store.ts` 是业务写入中心，负责数据加载、校验、创建、更新、删除、导入、导出和落盘。
3. `src/importExport/dataMigration.ts` 负责数据迁移 JSON 的构建、压缩、解析、展开、摘要和引用重映射。
4. `src/importExport/formattedTasks.ts` 负责两类 Markdown：今日完成极简 Markdown 和新任务计划复杂 Markdown。
5. `src/views/compositeDisplay.ts` 负责组合任务在视图层的展示聚合，今日任务页和任务总览共用同一口径。
6. `src/components/compositeTaskCards.ts` 负责组合任务轻量项、挂入真实子任务和弹窗子任务列表的统一卡片展示。
7. `src/views` 负责页面渲染，`src/components` 负责模态框、建议框和导入对话框。

## 设计原则

1. 数据迁移 JSON 是唯一完整迁移格式。
2. 今日完成极简 Markdown 只完成今天已有任务。
3. 新任务计划复杂 Markdown 只负责创建或覆盖任务计划。
4. 表格、看板、甘特图、思维导图不保存独立副本，全部回写同一条 `Task`。
5. 文档只描述当前实现，不保留历史迁移包或旧接口说明。
