# 01. 项目概述

> 本文档中的历史切片、完整迁移包和周任务图详细图表述已过时。当前实现以 `02-数据存储与模型.md`、`07-同步与边界规则.md` 与 `17-明文导出与周任务图简化.md` 为准。

本插件用于在 Obsidian 中提供项目管理、周任务安排、今日聚焦、重复任务管理和完成热度统计能力。

## 1. 当前产品定位

当前版本已经不再是最初的单次任务 MVP，而是升级为以下形态：

1. 任务总览页固定在主内容区作为独立 ItemView 打开。
2. 总览页重点展示热度图、周任务图和最近 30 天趋势图。
3. 今日任务页保持轻量，专注处理当天发生的任务。
4. 项目进度页负责项目级任务集中管理。
5. 任务模型支持单次、每日重复、每周此时重复和自定义日期集合；当前弹窗主要暴露前三种，格式化文本导入可使用自定义日期集合。
6. 任务支持普通任务与组合任务两种结构。
7. 项目管理页支持表格、看板、甘特图、思维导图四视图联动。
8. 快速记录页支持写日记、追加任意 Markdown 笔记、快速创建任务，以及向思维导图节点追加正文或评语节点。
9. 全局 Markdown 项目任务块可后台扫描并导入任务缓存。
10. 今日页和周任务图可从实例上下文编辑某一次发生的标题、描述和时间。
11. 支持“导出全部记录”为明文 Markdown 任务清单，并可通过快速记录-创建任务跨设备创建或覆盖任务。

## 2. 核心页面

### 2.1 任务总览页

任务总览页是用户进入插件后的主工作台。

它包含：

1. 热度图。
2. 周任务图。
3. 最近 30 天趋势图。
4. 总体任务统计卡片。

### 2.2 项目管理页

项目管理页用于按项目维度查看任务，并把重复任务按“当前活动切片 + 历史切片”集中管理。

它包含：

1. 表格视图。
2. 看板视图。
3. 甘特图视图。
4. 思维导图视图。

### 2.3 今日任务页

今日任务页只关心“今天该做什么”，展示的是当天发生的任务实例。

### 2.4 快速记录页

快速记录页固定在右侧侧边栏，用于快速输入内容。

它支持：

1. 写入每日日记。
2. 追加任务笔记。
3. 从格式化文本创建任务。
4. 把多行文本追加为思维导图评语节点，或追加到任务 / 评语正文。

### 2.5 热度图

热度图按“实际完成日期”统计用户近 12 个月的任务完成情况。

## 3. 数据思想

当前版本采用“系列切片存储 + 运行时展开实例”思路：

1. 未发生历史切片前，每个重复任务保存一条 Task。
2. 当任务被修改且包含过去发生日期时，Store 会保留过去切片并生成今天/未来活动切片。
3. `occurrenceDates` 记录这条任务切片会在哪些日期发生。
4. `occurrenceStates` 记录每次发生的完成状态；普通任务记录完成时间，组合任务额外记录已完成轻量检查项集合。
5. `occurrenceOverrides` 记录某次发生的时间或标题覆盖。
6. `viewState` 保存看板、甘特图、思维导图的视图状态。
7. `sourceLinks` 保存任务与笔记、快速记录、手动来源的关系。
8. `mindmapComments` 保存只属于思维导图的评语节点树。
9. 今日页、周任务图、热度图都基于运行时展开后的实例来展示。
10. 组合任务在实例层按轻量检查项和挂入该组合任务的子任务共同计算展示进度。

## 4. 设计原则

1. 录入效率优先于复杂装饰。
2. 任务规则统一，页面展示可以不同。
3. 重复任务在项目进度页必须保持单行，避免同一任务被拆成十几条。
4. 不兼容旧数据结构，直接以新模型为准。
5. 表格、看板、甘特图、思维导图不能各自保存任务副本。
6. 历史记录优先，禁止任何自动顺延或迁移型逻辑改写过去日期。
