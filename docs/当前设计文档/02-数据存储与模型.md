# 02. 数据存储与模型

当前数据默认存放在 Vault 内 `project-manager-data/`。

```text
project-manager-data/
├── config.json
├── projects.json
├── project-pages.json
├── note-task-index.json
├── write-history.json
└── tasks/
    ├── 2026-05.json
    └── 2026-06.json
```

## 配置模型

`PluginConfig` 保存插件配置：

1. 视图名称：`overviewTab1Name`、`overviewTab2Name`、`dialogTabName`。
2. 数据目录：`dataFolder`。
3. 今日与排程默认值：`timeSlotMinutes`、`showCompletedTasks`、`defaultTaskDurationMinutes`、`defaultTaskStartTime`。
4. 快速记录写入位置：`dailyNoteFolder`、`dailyNoteDateFormat`、`dailyNoteMode`、`dailyNoteSingleFilePath`。
5. 快速记录默认目标：`defaultDialogTarget`。
6. Markdown 插入头：`noteAppendHeader`、`dailyNoteHeader`。

## 项目与项目页

`Project` 保存项目名称、描述、颜色、状态和时间戳。创建项目时同步创建一条 `ProgressPage`，用于保存项目页名称和表格列顺序。

删除项目会删除对应项目页和该项目下的任务。项目页排序保存在 `project-pages.json`。

## 任务模型

`Task` 是唯一真实任务记录：

```ts
type Task = {
  id: string;
  kind: "simple" | "composite";
  title: string;
  description?: string;
  projectId?: string;
  status: "todo" | "doing" | "blocked" | "done";
  priority?: "low" | "medium" | "high" | "urgent";
  tags: string[];
  date: string;
  startTime?: string;
  endTime?: string;
  recurrence: "once" | "daily" | "weekly" | "custom";
  recurrenceCount?: number | null;
  recurrenceUntil?: string | null;
  occurrenceDates: string[];
  occurrenceStates: TaskOccurrenceState[];
  occurrenceOverrides: TaskOccurrenceOverride[];
  subtasks: TaskSubtask[];
  viewState: TaskViewState;
  sourceLinks: TaskSourceLink[];
  notes: TaskNote[];
  mindmapComments: TaskMindmapComment[];
  createdAt: string;
  updatedAt: string;
  revision: number;
};
```

`occurrenceDates` 是任务实例日期集合。页面层会把一条 `Task` 展开为多个 `TaskOccurrence`。

时间字段只接受 `HH:mm`，取值范围为 `00:00` 至 `23:59`。开始时间和结束时间必须同时填写，且结束时间必须晚于开始时间；不符合格式的输入会在 `ProjectManagementStore` 写入入口被拒绝。

## 组合任务

组合任务使用同一份 `Task` 模型表达：

1. `kind: "composite"` 表示任务是组合容器。
2. `subtasks` 保存轻量组合项。轻量项只有标题、可选开始时间、可选结束时间和排序，不是独立任务记录。
3. `viewState.mindmap.parentTaskId` 指向某个组合任务时，当前任务是挂入该组合任务的真实子任务。真实子任务保留自己的重复规则、完成状态、来源链接、笔记和视图状态。
4. 视图层通过 `src/views/compositeDisplay.ts` 聚合同一天的组合父任务、轻量项和真实子任务。

组合任务必须填写开始时间和结束时间。带时间的轻量项必须落在组合任务时间范围内。挂入组合任务的真实子任务必须填写开始时间和结束时间；它的每个发生日期必须存在对应的组合父任务实例，且子任务时间必须落在对应父任务实例时间范围内。

同一组合容器内的父任务、轻量项和真实子任务共享容器时间，不互相触发自动排程冲突；组合容器外的任务仍按同日时间段互斥。

## 完成状态与单次覆盖

`TaskOccurrenceState` 只保存已完成或部分完成的实例。普通任务只要存在对应日期状态即为完成；组合任务通过 `completedSubtaskIds` 计算轻量项进度。挂入组合任务的真实子任务使用自己的 `TaskOccurrenceState` 计算进度，再由视图层汇总到组合父任务展示。

`TaskOccurrenceOverride` 保存重复任务某一次实例的标题、描述、开始时间、结束时间、跳过标记和原因。展开实例时优先使用覆盖字段。

## 视图状态

`TaskViewState` 保存表格以外的视图状态：

1. `board`：看板列和排序。
2. `gantt`：甘特图排序、依赖、锁定和里程碑。
3. `mindmap`：思维导图父级、排序、展开和坐标。

思维导图评语保存在 `mindmapComments`，可以挂在任务节点下，也可以挂在另一条评语下。

## 写入规则

所有业务写入都经过 `ProjectManagementStore`。新增、编辑、删除、完成状态变化会写入对应 `tasks/YYYY-MM.json`，然后回读磁盘并触发视图刷新。写入历史保存在 `write-history.json`。
