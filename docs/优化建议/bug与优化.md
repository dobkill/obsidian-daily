## 0. 全局 UI 重构提示词

```text
你现在要优化一个 Obsidian 项目进度管理插件的前端 UI。请先阅读当前代码结构，识别页面组件、样式文件、状态管理、数据模型和路由/Tab 结构，不要急着改代码。

目标：在不破坏现有功能、数据结构和交互逻辑的前提下，把当前 UI 优化成更高级、更统一、更适合 Obsidian 深色主题的产品级界面。

整体设计方向：
1. 使用深色 Obsidian 风格：背景以 #0f1117、#141722、#1b1f2b 为主。
2. 主强调色使用紫色：#8b5cf6、#7c3aed、#a78bfa。
3. 辅助状态色：
   - 完成：绿色 / 青绿色，如 #34d399、#10b981
   - 进行中：蓝紫色，如 #6366f1
   - 待办：蓝色或紫色
   - 阻塞：橙红色，如 #f97316
   - 删除 / 危险操作：红色，如 #ef4444
4. 全局统一圆角、边框、阴影、卡片风格：
   - 卡片圆角 14px - 20px
   - 边框使用 rgba(148, 163, 184, 0.16)
   - 背景使用轻微渐变或半透明玻璃感
   - 避免大面积纯黑，使用层级深浅区分模块
5. 统一字体层级：
   - 页面标题：24px - 28px，font-weight 700
   - 区块标题：18px - 20px，font-weight 650
   - 卡片标题：15px - 16px，font-weight 600
   - 辅助说明：12px - 13px，颜色降低透明度
6. 所有按钮、Tab、Chip、状态标签、进度条样式必须统一。
7. 页面左右上下间距要更舒展，避免当前 UI 的拥挤感。
8. 不要修改用户已有数据、任务逻辑、重复任务逻辑、项目统计逻辑。
9. 不要大规模重写业务代码。优先重构样式、布局组件、可复用 UI 组件。
10. 如果项目中已有组件体系，请复用；如果没有，请抽象以下基础组件：
    - MetricCard
    - PanelCard
    - StatusBadge
    - ViewTabs
    - ProgressBar
    - IconButton
    - EmptyState
    - TaskCard
11. 所有 UI 文案保持中文。
12. 需要兼容 Obsidian 插件环境，不要引入过重依赖。除非项目已经使用 React/Vue/Svelte，否则不要强行更换技术栈。

请先输出你的改造计划，包括：
- 你识别到的主要页面和组件
- 你准备修改的文件
- 你准备抽象的公共组件
- 你会保留不动的业务逻辑

然后开始实施 UI 优化。完成后请说明：
- 修改了哪些文件
- 每个页面改善了什么
- 是否影响现有功能
- 是否有需要我手动检查的地方
```

## 1. 任务总览页优化提示词

```text
请优化「任务总览」页面 UI。

当前页面包含：
- 顶部标题：任务总览
- 说明文案：热度图、周任务图和近 30 天趋势等统计
- 指标卡片：今日待办、今日完成、本周任务、项目数
- 热度图
- 周任务图
- 最近 30 天任务趋势折线图
- 右上角 Tab：任务总览 / 项目进度

优化目标：
把这个页面做成一个清晰、高级、信息密度合理的仪表盘，而不是简单堆叠模块。

具体要求：

1. 页面整体布局
- 左侧保留或新增窄侧边栏导航，如果项目已有侧边栏，请统一高亮当前页面。
- 主内容区使用 max-width 或自适应布局，不要贴边。
- 顶部 Header 左侧显示：
  - 标题：任务总览
  - 副标题：热点项目、周度任务与近 30 天趋势，帮助你掌握全局进度。
- 顶部右侧放置两个切换按钮：
  - 任务总览：当前激活
  - 项目进度：非激活
- Header 与下方内容之间增加清晰间距。

2. 指标卡片区域
- 将「今日待办」「今日完成」「本周任务」「项目数」做成 4 个统一风格的 MetricCard。
- 每张卡片包含：
  - 圆形图标区域
  - 指标名称
  - 大号数字
  - 单位，如「任务」「个」
- 卡片背景使用深色渐变，边框轻微发光。
- 数字要比标题更突出。
- 图标颜色分别区分：紫色、绿色、蓝色、紫蓝色。

3. 热度图区域
- 热度图放在指标卡片右侧或下方的大型 PanelCard 中。
- 标题：热度图
- 右上角增加筛选器：最近 12 个月
- 月份标签、星期标签、活动方块对齐要更整齐。
- 方块尺寸统一，间距统一，圆角 4px。
- 增加活动强度图例：「较少」到「较多」。
- 空数据或低活跃状态不能显得脏乱，默认灰色块透明度降低。

4. 周任务图区域
- 标题：周任务图
- 副标题显示周范围，例如：2026-05-25 至 2026-05-31。
- 右上角有上一周、本周、下一周按钮。
- 7 列卡片按周一到周日展示。
- 每一列是一个 DayColumn：
  - 顶部显示星期和日期
  - 右上角有「+ 新增」按钮
  - 空状态显示图标和提示，例如「暂无任务」「享受一个高效的开始吧」
- 任务卡片 TaskCard 需要优化：
  - 标题突出，如「学习」
  - 显示时间、项目、重复规则、次数
  - 显示子任务完成度，如 0/1 子任务
  - 子任务用嵌套小卡片展示
  - 完成状态用绿色背景或绿色边框
  - 待办状态用紫色 / 橙色左边线
- 当前选中日期或今天要有轻微高亮，但不要太刺眼。

5. 最近 30 天任务趋势
- 用一个大卡片承载折线图。
- 标题：最近 30 天任务趋势
- 左上角图例：
  - 新建任务：紫色
  - 完成任务：绿色
- 右上角筛选器：近 30 天
- 图表背景要干净，网格线降低透明度。
- 折线使用圆点节点，末端数据高亮。
- X/Y 轴字号更小但清晰。
- 如果图表组件已有实现，只优化样式；不要改统计逻辑。

6. 响应式
- 宽屏下：指标卡片 + 热度图可以横向布局。
- 窄屏下：自动堆叠。
- 周任务图在窄屏下允许横向滚动，而不是压缩到不可读。

验收标准：
- 页面视觉层级明显提升。
- 卡片、按钮、Tab、Chip 风格统一。
- 信息比原来更清晰，不拥挤。
- 没有破坏统计数据。
- 深色主题下文字对比度足够。
```
