# 状态  作用修改：
   子任务 和 普通任务: 
         若状态为 “阻塞” 或 “代办” ，即任务若超时未完成，则不能消费剩余重复次数 ，且 不显示在 今日任务 页 的 未完成 列表中
         例如：
            任务为重复10次（执行足额次数 false），状态为 阻塞 or 代办，若今日未完成，则不能消费剩余重复次数
                                                                 ，若今日完成，剩余执行次数-1
            任务为重复10次（执行足额次数 false），状态为 进行中，若今日未完成 或者 完成，剩余执行次数都必须 -1
            任务为重复10次（执行足额次数 true），无论何种状态，只有完成任务，剩余执行次数才能-1

# 新增任务弹窗：
   【1】重复规则 修改： 
         重复类型：
            增加：每月此时重复 （含义是每个月的此时重复一次，例如：若 任务的 时间安排是 2026-06-25 重复10次， 则 2026-07-25， 2026-08-25，，， 重复10次）
            增加：永久重复（本质是 每日此时重复 +重复次数为 int 最大值  的快捷选择）
   【2】重复规则 修改：
            每日重复 ，改名为 每日此时重复

   【3】重复规则 修改：
      核心本质：
         （1）重复规则最核心的：
                 1剩余执行次数 ， 2重复类型 ，3是否执行足额次数，4重复结束日期
         （2）单次任务 = 每日此时重复 + 重复次数为1 
                  单次任务 本质是 快捷选择标签 
         （3）永久重复 = 每日此时重复 + 重复次数为 int 最大值
                  永久重复 本质是 快捷选择标签
         （3）重复类型：本质就 3个 ，分别是：每日此时重复，每周此时重复，每月此时重复
    
   【4】重复规则 修改：
             增加 switch button
             此button 默认关闭，用户需要手动开启
             button 命名为： 执行足额次数
             含义：
               （1）若用户开启了 此button ，且任务为重复10次，代表着 任务必须被完成 10次  若昨日未完成 则 任务不可被计入消费
               （2）若用户关闭了 此button ，且任务为重复10次，无论昨日 任务是否 完成，都可被计入消费
            补充：
               若任务为阻塞 或 代办状态，所有任务都必须延期，不能被消费剩余重复次数
   
   【5】重复结束日期：
         若填写重复结束日期，任务在重复结束日期后，属于完成 状态 ，剩余执行次数为 0
         不填写，则无时间约束
         默认值：不填写
